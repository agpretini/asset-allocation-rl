# Experiments

This directory stores reproducible experiment snapshots.

Each experiment folder is generated by the experiment tracking utilities in
`src/experiments/`. Folders are numbered automatically and should not be edited
by hand unless you are adding explanatory notes.

## Folder Naming

Current prefixes:

- `q_learning_###`: tracked Q-learning evaluation runs.
- `hpo_###`: hyperparameter optimization runs.

Examples:

```text
experiments/
├── q_learning_001/
├── q_learning_002/
├── hpo_001/
└── hpo_002/
```

## Standard Contents

Each experiment folder contains:

- `config.yaml`: configuration and assumptions used for the run.
- `metrics.json`: headline metrics for the run.
- `actions.csv`: selected actions or action-related diagnostics.
- `portfolio_values.csv`: portfolio value path, when available.
- `notes.md`: short human-readable notes.

## How To Generate Experiments

Track the current Q-learning evaluation:

```bash
uv run python scripts/evaluate_results.py
```

Run hyperparameter optimization:

```bash
uv run python scripts/optimize_hyperparameters.py
```

## Relationship To `data/processed`

`data/processed/` contains reusable generated artifacts organized by producer.

`experiments/` contains reproducible snapshots that bundle configuration,
metrics, paths, actions, and notes for a specific run.
